# JSON style

Covers `versions.json` files and other JSON data files in the corpus.  These files are **generated by jq**, not written by hand — they exist as the authoritative machine-readable state for Dockerhub image versions, build parameters, etc.

## Indentation

**2 spaces per level.**  No tabs in JSON files.  This matches jq's `--tab` option output when `--tab` is **not** used — jq's default pretty-printer uses 2 spaces, and that default is what produces these files.

## Key ordering

Object keys are **sorted alphabetically**.  jq's `-S` / `--sort-keys` flag is always passed when writing JSON files:

```bash
jq <<<"$json" -S . > versions.json
```

This makes `git diff` output meaningful and deterministic.

## Root type

The root value of `versions.json` files is always an **object**, not an array.  Top-level keys are version strings:

```json
{
  "10.0": {
    "url": "https://download.qemu.org/qemu-10.0.tar.xz",
    "version": "10.0"
  },
  "10.1": {
    ...
  }
}
```

## Value types

- **Strings** — double-quoted
- **Numbers** — unquoted
- **Booleans** — `true` or `false`, unquoted
- **Null** — `null`, but rare; absent fields are preferred over explicit `null`
- **Arrays** — block style with one element per line when multi-element
- **Objects** — block style; compact `{}` only for genuinely empty objects

## Absent vs null

Fields that are optional are **absent entirely** from the JSON object when not applicable.  Explicit `null` values are rare — the jq code that generates these files uses `// empty` or conditional filtering to omit fields rather than setting them to null.

## Generation pattern

The standard pattern for generating a versions file:

```bash
json='{}'                                  # start with empty object
# ... populate with jq expressions ...
jq <<<"$json" '.' > versions.json          # pretty-print (2 spaces, unsorted) OR
jq <<<"$json" -S . > versions.json         # pretty-print with sorted keys
```

Some scripts use a final `.` pass (identity) to ensure proper formatting even when the intermediate JSON was built with `--compact-output`.

## Notable omissions

- Trailing commas — JSON does not allow them, and jq does not produce them
- Comments — JSON does not support comments; they never appear in `.json` files
- Tab indentation — always 2 spaces
- `null` as a sentinel value for "missing" — fields are omitted instead
- Unsorted keys — `-S` is always used for output files
- Numbers with unnecessary precision — integer counts are integers, not `1.0`
